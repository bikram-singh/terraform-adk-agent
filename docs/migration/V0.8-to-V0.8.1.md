# Migration: v0.8 to v0.8.1

Version 0.8.1 replaces the global generated-file whitelist with a
plugin-owned file policy.

Each generator declares its files through:

    ServiceMetadata.generated_files

The intelligence engine verifies that emitted files exactly match the plugin
metadata before creating a workspace.

The secure file writer still enforces:

- No absolute paths
- No nested paths
- No hidden files
- Safe filename characters
- Approved Terraform and documentation file types
- Workspace containment
- Explicit overwrite behavior

The engine owns `validation-report.md`.

Existing GCS, Cloud Run, GKE, MCP, and Registry Service behavior remains
compatible.
