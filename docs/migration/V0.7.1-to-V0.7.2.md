# Migration: v0.7.1 to v0.7.2

## Purpose

Prevent raw Terraform Registry MCP documentation from flowing directly into
the model context.

## Architecture change

Before:

    ADK model -> raw MCP tools -> raw Registry markdown -> model

After:

    ADK model -> Python Registry service -> MCP -> sanitizer -> structured dict

## Benefits

- Removes active URI payloads before model consumption.
- Bounds Registry content size.
- Keeps the existing Registry-only MCP scope.
- Avoids exposing HCP Terraform mutation tools.
- Provides stable application-level tool names.
- Isolates future MCP schema changes behind one service layer.

## Compatibility

GCS and Cloud Run generator tools remain unchanged.

MCP remains Docker-based and Registry-only.

## Validation

Run:

    python -m pytest tests/unit/test_registry_service.py -v
    python -m pytest tests -v

Optional live test:

    $env:RUN_TERRAFORM_MCP_INTEGRATION = "true"
    python -m pytest tests/integration/test_registry_service_live.py -v
